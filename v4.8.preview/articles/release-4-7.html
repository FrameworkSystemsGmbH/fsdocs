Hinweise

  • Die PBE in FS wurde aktualisiert.
  • Die ausgelieferte JRE unterstützt keine ARM-Prozessoren. Dies ist auch in den Requirements dokumentiert.

Neuerungen

  • Events klickbarer Links im JxBrowser, die durch dynamische Änderungen des Doms nach dem ersten Laden hinzugefügt wurden, werden nun auch korrekt gefeuert.

Korrigierte Fehler

  • 2025047832 - Publish2Go: Bei einer Datenbank-Connection kann es, z.B. im Publish2Go, zu einer Fehlermeldung kommen: System.TypeLoadException: Error loading provider assembly.

  • 2025057952 - Package-Manager: Wenn beim Export einer Package-Version die Checkbox Include Base Packages gesetzt ist und eine base Package-Version ein HotFix ist, wird für diese sowhol SR, als auch Full Package Export angeboten. Der Export ist aber immer ein Full-Export.

  • 2025068000 - Package-Manager: Wird eine Package-Version, welche sich im SR-Modus befindet, mit dem Compile-Wizard compiliert, dann kann es passieren, dass Saved Codes nicht vollständig erzeugt werden. In der Folge kommt es in Customizing-Packages zu Compile-Fehlern.

    Note

    Im SR-Modus werden jetzt immer alle Namespaces compiliert, damit ggf. fehlende Saved Codes erzeugt werden. Dadurch kann es höhere Compile-Zeiten geben.

  • 2025068015 - Package-Manager: Wird eine Package-Version exportiert und ein Basis-Package befindet sich im Hotfix-Modus, dann stimmt in der Export-Datei die Package-Hierarchie nicht. Beim Package-Import kommt es dann zu Fehlermeldungen.

  • 2025057859 - IDE: Wird eine Component kopiert, bei der in einem Property Control-Types zugeordnet sind, dann lässt sich die eingefügte Kopie nicht speichern.
    Es wird eine SQL-Fehlermeldung angezeigt: Der Wert NULL kann in die CTFCP_CT_ID-Spalte, dbo.tblRep_ControlTypeForCP-Tabelle nicht eingefügt werden.

  • 2025057958 - IDE: Beim CheckOut kann es zu einer UnhandledException kommen, wenn im Hintergrund Code für die CodeAnalysis/Intellisense generiert wird. Dies tritt nur in seltenen Fällen auf, wenn sich Aufrufe des asynchronen Codeeditors mit dem synchronen Framework Studio überschneiden.

  • 2025047833 - Data Provider: Bei Insert und Update SQL-Statements, welche von FrameworkStudio generiert werden, werden keine Parametergrößen angegeben. Dadurch entstehen sehr viele unterschiedliche Statements, die vom SQL-Server nicht wiedererkannt werden. Dies führt zu Performanceverlusten.

  • 2025057919 - HtmlClient: Bei der MobileClient App auf einem Android 10 Betriebssystem kommt es zu Layoutfehlverhalten, wenn durch Klick auf ein Editfield die Tastatur geöffnet wird. Ein weißer Balken zeichnet sich über der Tastatur und verkleinert den Arbeitsbereich der App extrem.

  • 2025057942 - Package Merge: Bei Form-Dokumentationen und Dokumentationen wird beim PackageMerge die Target-Dokumentation mit der Source-Dokumentation überschrieben. Es findet also kein Merge der Dokumentation statt.

Hinweis

Es gibt einen neuen Artikel zum Thema "DevExpress Exception ScriptItemizeOpenType in Usp10.dll nicht gefunden" in unserem FAQ Bereich. Dieser Fehler wurde uns nun häufiger gemeldet, deshalb möchten wir Sie bitten unsere derzeitigen Requirements mit Ihren Systemen zu vergleichen. Testen und führen Sie notwendige Updates durch, falls dies möglich ist.

Korrigierte Fehler

  • 2025037564 - IDE: Werden Component- oder DataContract-Properties gelöscht, welche in grafischen Transformationen zugeordnet sind, dann fällt diese Zuordnung ohne weiteren Hinweis auf die darüberliegende Ebene zurück. Infolgedessen erzeugt die Code-Generierung fehlerhafte Zuordnungen, welche zur Runtime zu einer InvalidCastException führen können.

    Note

    Diese Zuordnungen erzeugen jetzt fehlerhaften Code, der zu Compile-Errors führt. Beim Öffnen der betroffenen Transformation im Designer werden die Fehler angezeigt und fehlerhafte Zuordnungen gelöscht.

  • 2025037643 - IDE: FS Lizenzen gelten als ungültig, wenn diese ohne Datum von/bis ausgestellt werden. Die Angabe einer Version reicht nicht.

  • 2025047734 - PBE: In Framework-Studio 4.6 wurde beim Export der Dokumentation ein automatischer Ordner bestehend aus Datum, Package, Version und Iso-Code eingeführt. PBE erzeugt seinerseits bereits einen ähnlichen Ordner. Das führt zu einem doppelten Ordner und unnötig langen Datei-Pfaden.

    Note

    Die aktualisierte Version der PBE.exe liegt im Framework Studio Programm-Verzeichnis im Unterordner PBE.

  • 2025047708 - Startpage: Die Auswertung der Startpage-URL erfolgt case-sensitive. Wenn das Wort "start" in der URL ".../FSDemoApplication/start/?lang=de" nicht klein geschrieben wird, wird der Fehler "Error while receiving data from broker!" im Browser angezeigt.

Korrigierte Fehler

  • 2025027380 - Java Client: Wenn an einer Registerkarte das Event OnFirstSelect implementiert ist, dann kann es passieren, dass bei der ersten Eingabe in einem TextFeld das OnLeave-Event nicht korrekt gefeuert wird.

  • 2025017265 - Java-Client: Wenn ein Form, das über ein Event geschlossen werden soll, im selben Zug fokussiert wird, kommt es zu einer Fehlermeldung.

  • 2025027375 - Package-Manager: In einem Oracle-Repository kann es passieren, dass das Update einer Package-Version zu einem Fehler führt. Im Script 4.6.0.0 2022.11.18 DeprecatedLayoutCheck.ora wird eine Sql-Exception ausgegeben FS.DataClient.FrameworkDbException: ORA-00932: Inkonsistente Datentypen: CHAR erwartet, NUMBER erhalten.

    • Der Exception-Editor erzeugt einen mehrzeiligen Code, der Compile-Fehler produziert und der Exception-Editor erkennt diesen Code nicht korrekt.

  • 2025027305 - Framework: Die Methode LoadCount() liefert bei GroupBy-Components falsche Werte zurück. Die Anzahl der Sätze wird in einer Gruppierung ermittelt.

Neuerungen

Siehe Neuheiten Framework Studio 4.7.

Korrigierte Fehler

In der Version 4.7 sind alle Fehler-Korrekturen bis einschließlich Framework Studio 4.6.12 enthalten. Zusätzlich wurden die folgenden Bugs behoben.

  • 2024116805 - Publish / Runtime-Lizenzen: Wird im Basis-Package eine neue AccessUnit angelegt und die Lizenz-Datei nicht aktualisiert, dann enthält die Lizenz-Datei keine Information über diese AccessUnit. Wird in einem darauf aufsetzenden Customizing-Package eine Lizenz angelegt, und diese AcessUnit freigeschalten, dann hat die Anwendung Zugriff auf diese AccessUnit.
    AccessUnits müssen aber vom jeweiligen Package freigegeben sein, damit man Zugriff darauf erhält.

    Note

    Mit Version 4.7 wird die Lizenz direkt im Broker "aktualisiert" und ausgewertet. Bei Änderungen am AccessUnit-Baum müssen Lizenzen deshalb nicht mehr zwingend aktualisiert werden. Siehe auch Neuheiten

  • 2024025119 - IDE: Der Name eines CodeFiles darf . als Sonderzeichen enthalten, dies kann später zu Folgeproblemen führen.

    Note

    Der Name muss ein gültiger C#-Identifier sein.
    Beim Compile werden jetzt entsprechende Warnings erzeugt und im Designer wird ein ungültiger Wert gekennzeichnet.

  • 2024076125 - Code Generierung: Ein ServiceHost mit aktiver Option "Generate REST Endpoint" generiert die Warning FSWarn[2033] At least one endpoint has to be defined in order to use this service host. Diese Warning ist nicht notwendig, da ein Endpoint automatisch angelegt wird.

  • 2024096503 - Code-Generierung: ist einem alten Deprecated Control, das nicht in das neue Layout übernommen wurde (z.B. einem Label), ein Event zugeordnet, dann wird der Aufruf noch immer in den Code generiert. Zudem kann die Event-Methode nicht gelöscht werden, weil die Event-Zuordnung als Dependency erkannt wird.

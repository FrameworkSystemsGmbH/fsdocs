Domain Authentifizierung

Framework Studio Anwendungen unterstützen die Benutzer-Authentifizierung mithilfe der Windows Authentifizierung über Waffle und das Active Directory.

Ablauf einer Authentifizierung

Für den Anwender läuft die Anmeldung folgendermaßen ab:

  1. Start der Anwendung
  2. Broker, Client und Auth-Service authentifizieren den Benutzer im Hintergrund.
  3. Die Anwendung startet mit angemeldetem Benutzer.
Detaillierte Beschreibung

  • Der Java-Client startet und meldet dem Broker im ersten Request dass er Domain Authentifizierung kann.
  • Der Broker fragt beim Authentication Service nach Domain und Domain-Controller und meldet diese an den Java-Client zurück.
  • Der Java-Client ermittelt mit den Informationen über Waffle einen Windows AuthToken.
  • Der AuthToken wird über den Broker zum Auth-Service übermittelt, welcher den AuthToken über Waffle validiert und eine gültige Windows Identität zurückgibt.
  • Aus dieser Identität wird ein gültiger XML-Token erstellt. Mit diesem XML-Token kann die Anwendung den Benutzer identifizieren.
  • Die Anwendung startet mit dem angemeldeten Benutzer.

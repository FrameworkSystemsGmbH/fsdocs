Update auf Version 4.8

Dieser Leitfaden stellt dar, wie ein Update auf die Version 4.8 durchgeführt wird.

Vorbereitung

Zunächst müssen alle Updates von LTS 4.5, 4.6 und 4.7 durchgeführt werden. Dazu können Sie folgende Checkliste nutzen:

1. Update auf Version 4.5

OpenID Connect - Update: Überprüfen Sie bitte, ob Sie bereits die OpenIdConnect Änderungen durchgeführt haben.

  • Weitere Neuheiten und Breaking Changes - Neuheiten FS 4.5

2. Update auf Version 4.6

Deprecated Layout entfernt: Prüfen Sie, ob Sie alle Forms vollständig migriert haben.

  • Weitere Neuheiten und Breaking Changes - Neuheiten FS 4.6

3. Update auf Version 4.7

Important

Führen Sie den Update-Leitfaden für die 4.7 in genau der Reihenfolge durch, wie es beschrieben ist!

Tip

Mit der Einführung der Dependency Injection in FS 4.8 mussten wir den neu eingeführten GlobalContext aus FS 4.7 überarbeiten. Beim Update auf FS 4.7 empfehlen wir deshalb die vorherige Kenntnisnahme der Änderungen in FS 4.8.

  • Weitere Neuheiten und Breaking Changes - Neuheiten FS 4.7

4. Crystal Reports abgekündigt

Stellen Sie alle Reports von CrystalReports auf DevExpress um. Nach einem Update können Crystal Reports in der Anwendung nicht mehr gedruckt werden.

Nach dem FS Version Upgrade

Nachdem Sie auf die 4.8 upgedated haben, beachten Sie bitte folgende Neuheiten:

  1. Update Hinweise zu Dependency Injection: Führen Sie die Umbaumaßnahmen bezüglich Dependency Injection durch.
  2. FxBrowser entfernt: Der Workaround und Fallback auf den alten, obsoleten Browser ist entfernt worden.
  3. Update auf Java 21: Eigene Custom Controls müssen mit Java 21 neu kompiliert werden.

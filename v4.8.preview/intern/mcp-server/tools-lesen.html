Tools

Alle Tools des MCP-Servers arbeiten read-only — es werden keine Änderungen am Repository vorgenommen. Die Ergebnisse sind auf das aktive Package und dessen Abhängigkeiten beschränkt.

Tip

Die vollständige Parameterbeschreibung jedes Tools wird vom MCP-Server über tools/list bereitgestellt. KI-Clients erhalten diese automatisch bei der Verbindung.

Umgebung

Tool Beschreibung
fs_get_environment Aktives Package, Domain, Verbindungsinformationen

Elemente auflisten

Tool Beschreibung
fs_list Alle Elemente nach Typ auflisten. Unterstützte Elementtypen: component, form, service, modular_component, namespace, package, db_table, metadatatype, global_object, datasource, global_event, service_contract, service_host, workflow, report, access_unit, data_contract. Parameter: elementType, namespaceName (optional), offset + limit (Pagination).

Elemente lesen

DevObjects (Component, Form, Service, Modular Component)

Tool Beschreibung
fs_get_devobject Details auf DevObjects (Component, Form, Service, Modular Component) abrufen. Parameter: elementType (component|form|service|modular_component), fullName, detail (optional: summary|full, default full), method (optional: nur diese Methode mit vollem Code), property (optional: nur diese Property). Detail-Level: summary zeigt Struktur (Property-Namen/Typen, Methoden-Signaturen) ohne Code-Bodies — ideal für Orientierung großer Komponenten. full gibt komplettes YAML mit allen Code-Bodies (Standard).

Andere Elementtypen

Tool Beschreibung
fs_get Details auf andere FS-Elemente abrufen. Parameter: elementType (package|db_table|metadatatype|data_contract|global_object|global_event|service_contract|service_host|workflow|report|datasource|access_unit), name.

Suche

Tool Beschreibung
fs_search_names Element-Namen und FullNames durchsuchen (alle Elementtypen)
fs_search_code Volltextsuche im Methoden-InnerCode über Components, Forms und Services. Parameter: pattern, regex (optional: false für Literal-Match, true für Regex-Pattern).
fs_search_dependencies Strukturelle Abhängigkeiten eines Elements finden (Properties, Typen, Vererbung)
fs_search_documentation MLDoc-Felder auf Metadatatypes, Properties und Forms durchsuchen
Note

fs_search_code durchsucht nur den Methoden-InnerCode, nicht Declarations oder Descriptions. fs_search_names durchsucht Element-Namen, nicht Property- oder Methodennamen innerhalb eines Elements.

Änderungshistorie & Vererbung

Tool Beschreibung
fs_get_element_history Versionsverlauf eines Elements (Datum, Benutzer, Kommentar)
fs_get_method_history Code-Änderungen einer Methode über verschiedene Versionen
fs_get_checkin_history Globaler CheckIn-Verlauf (wer hat wann was eingecheckt)
fs_get_inheritance_tree Vererbungsbaum eines Elements (alle Parent und Child Klassen)

Hinweise zur Nutzung

  • Für große Components (50+ Properties, 30+ Methoden) ist detail=summary effizienter als detail=full. Summary zeigt die Struktur ohne Code-Bodies.
  • Mit dem method=MethodName Parameter können Sie nur eine einzelne Methode mit vollem Code laden statt die ganze Component.
  • Mit dem property=PropName Parameter können Sie nur eine einzelne Property auslesen statt die ganze Component.
  • Collections sind Components mit ComponentType=Collection — sie erscheinen in fs_list mit elementType=component, nicht in einer separaten Liste.
  • fs_search_dependencies zeigt was ein Element referenziert — nicht welche Elemente das Element referenzieren. Für die Gegenrichtung fs_search_code verwenden.

Update JxBrowser für JavaClient

Das JxBrowser Control für den Java-Client wurde von Version 9.0.2 auf Version 9.3.0 aktualisiert. Das entspricht Chromium 150.

Release-Notes des Control-Herstellers: https://jxbrowser-support.teamdev.com/release-notes/2026/v9-3-0.html

Es sind keine besonderen Aktionen erforderlich. Mit dem Publish der Application wird die neue Version automatisch auf alle Clients verteilt.

Neuerungen

  • Im Runtime License Manager wurde die Tree-List, welche die Lizenzen darstellt, grafisch überarbeitet.
  • Der Dialog zur automatischen Generierung von DataContracts und dazugehöriger Transformationen aus Components wurde überarbeitet.
  • Das Output-Fenster wird nicht mehr automatisch im Vordergrund geöffnet, wenn dort neue Informationen ausgegeben werden. Stattdessen erhält die Registerkarte ein blaues Info-Image.

4.8.1 (03.07.2026)

Die Auslieferung umfasst die folgenden Pakete:

  • FS_4.8.1_Setup_x64
  • FS4.5.42_PrintService

Die Liste mit allen aktuellen Installations-Paketen finden Sie hier.

Updates

  • Die PBE wurde aktualisiert. Denken Sie daran diese Änderung in Ihre außerhalb von FS positionierten PBEs zu übernehmen.
  • Der HTML Client und die dazugehörige Mobile Client App wurden auf die Version 1.0.64 aktualisiert.

Neuerungen

  • Im Authentication and Authorisation Tool der Runtime Administration wurden die Grid-Controls überarbeitet.
  • Der HTML Client hat nun einen 15s Request-Timeout Mechanismus eingebaut.
  • Der Print-Server schreibt das Log-File nun täglich in eine neue Datei. Der konfigurierte Dateiname dient dabei als Basis; das aktuelle Datum wird automatisch als Suffix ergänzt (z. B. PrintServer_20260429.log). Ein Neustart des Dienstes beim Tageswechsel ist nicht erforderlich.
  • Der Print-Server verfügt nun über ein optionales Erweitertes Spooler Monitoring. Es überwacht den Windows Druckspooler aktiv und protokolliert den Lebenszyklus jedes Druckauftrags in zwei tagesrotierten Log-Dateien (SpoolerLog, FailedPrints). Bei erkannten Spooler-Fehlern wird die zugehörige FS-Transaktion automatisch als fehlgeschlagen markiert. Das Feature wird über ExtendedSpoolerLogEnabled in der PrintService.exe.config aktiviert. Weitere Informationen unter Erweitertes Spooler Monitoring.

Korrigierte Fehler

  • 2026069998 - Publish2Go: Wenn bei einem Publish2Go über die PBE eine Exception beim Schreiben der Applications oder ServiceHosts geworfen wird, wird diese nicht in das PBE Log übertragen.
    Der Publish2Go-Lauf wird als erfolgreich angezeigt, ist aber fehlerhaft.

  • 2026060001 - IDE: Wenn der Access Unit Search Dialog im Runtime License Manager mit dem X-Button geschlossen wird, lässt er sich nicht mehr öffnen.

  • 2026059822 - Services: CORS-Einstellungen beachten keine OPTIONS-Preflight-Requests.

Hinweise

  • Die in FS über den Menüpunkt Help aufrufbaren Dokumentationen Customization Conflicts und Package Merge wurden in die Online Dokumentation übertragen.

Update JxBrowser für JavaClient

Das JxBrowser Control für den Java-Client wurde auf Version 9.0.2 aktualisiert. Das entspricht Chromium 148.

Release-Notes des Control-Herstellers: https://teamdev.com/jxbrowser/releases/2026/v9-0-2/.html

Es sind keine besonderen Aktionen erforderlich. Mit dem Publish der Application wird die neue Version automatisch auf alle Clients verteilt.

Korrigierte Fehler

In der Version 4.8 sind alle Fehler-Korrekturen bis einschließlich Framework Studio 4.7.18 enthalten. Zusätzlich wurden die folgenden Bugs behoben:

  • 2025037568 - Data Provider: In bestimmten Konstellationen werden fehlerhafte OrderBy-Clauses ignoriert, anstatt eine Laufzeit-Fehlermeldung auszugeben (siehe SQL Parser).

    Note

    Es kann jetzt vorkommen, dass vermehrt Laufzeit-Fehlermeldungen auftreten.
    Das kommt auf die Definition der OrderBy-Clauses bzw. SQL-Statements in Ihrem Package an.

  • 2026039387 - Data-Provider: Bei der Analyse von DB-Differenzen wird versucht, in der Datenbank definierte "Default Values or Bindings" an Columns in FS-SQL zu parsen. Ist das SQL nicht FS-SQL kompatibel, führt dies zu einer ungewollten InternalParserException.
